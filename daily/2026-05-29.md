## Learning notes

### Core takeaway

今天我把区块链先理解成一个“很多人一起维护的公开记账本”。

一笔链上操作不是只发生在一个中心化服务器里，而是会被广播到网络中，由很多节点检查，通过共识算法确认后，被打包进区块，最后成为区块链上不可随便更改的一条记录。

### 1. blockchain / 区块链

区块链是一个不能随便篡改的公共账本。

它把一页页账单按顺序串起来，每一页叫一个“区块”，很多区块连在一起就是“区块链”。

### 2. network / 网络

网络是维护这本账本的一群电脑组成的系统。

比如以太坊网络，就是很多电脑一起运行以太坊规则。

### 3. node / 节点

网络里的每一台参与电脑都叫节点。

节点的工作是：

- 保存账本
- 检查交易
- 把信息传给其他节点

### 4. transaction / 交易

交易是你在链上发起的一次操作。

它不一定只是转钱，也可以是买 NFT、兑换代币、调用智能合约等。

### 5. block / 区块

区块可以理解为一批交易打包成的一页账单。

新区块会接在旧区块后面，形成连续记录。

### 6. 智能合约

智能合约是部署在区块链上的自动程序。

它像一台无人值守的自动售货机：规则提前写好，条件满足就自动执行，不能随便反悔。

### 7. Gas

Gas 是使用区块链时付给网络的“手续费”。

因为节点帮你计算、验证、记录交易，需要成本。操作越复杂，Gas 通常越贵。

### 8. DeFi

DeFi 是 Decentralized Finance，也就是去中心化金融。

简单说，就是不用传统银行或券商，也能在链上借钱、存钱、换币、赚利息的一类应用。

### 9. Layer2

Layer2 是建在主链上方的“加速层”。

主链像主干道，安全但容易堵；Layer2 像高架路，先在上面快速处理大量交易，再把结果提交回主链。

Layer2 常见目的：

- 更快
- 更便宜
- 缓解主链拥堵

### 10. 共识算法细节

共识算法解决的问题是：这么多节点，怎么决定“谁说的账本是对的”。

常见方式有：

- PoW，工作量证明：大家比拼计算能力，谁先算出答案，谁有机会记新区块。比特币用这个，安全但耗电。
- PoS，权益证明：谁抵押的币更多、表现更可靠，谁更有机会记新区块。以太坊现在用这个，更省能耗。
- BFT 类共识：一群验证者投票，只要足够多人同意，就确认结果。常用于速度要求更高的链或联盟链。

### One-sentence flow

> 你发起一笔 transaction 交易，它被广播到 network 网络，由很多 node 节点检查，经过共识算法确认后，被打包进 block 区块，最后成为 blockchain 区块链上不可随便更改的一条记录。

## Key concepts

1. 区块链：很多人一起维护的公开记账本。
2. 网络：运行某条链规则的一群电脑组成的系统。
3. 节点：网络里的参与电脑，负责保存账本、检查交易、传播信息。
4. 交易：链上的一次操作，不只包括转账，也包括买 NFT、兑换代币、调用智能合约等。
5. 区块：一批交易打包成的一页账单。
6. 智能合约：部署在链上的自动程序，条件满足就执行。
7. Gas：使用区块链时支付给网络的手续费。
8. DeFi：不用传统银行或券商，在链上完成金融活动的一类应用。
9. Layer2：主链上方的加速层，用来提升速度、降低成本。
10. 共识算法：节点之间决定“哪份账本是对的”的机制。
