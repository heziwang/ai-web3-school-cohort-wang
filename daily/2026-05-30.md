## Learning notes

### Core takeaway

今天我重点学习了 Web3 钱包与私钥安全。

我对 Web3 账户体系的第一层理解是：钱包不是“存币的软件”，而是管理私钥、生成地址、发起交易、连接 DApp 的工具。地址可以公开，私钥和助记词则代表账户控制权，绝对不能泄露。

### 1. 钱包是什么

Web3 钱包并不是用来真正“存放加密资产”的地方。资产记录在区块链账本上，钱包更像是用户管理账户权限的工具。

钱包的主要作用包括：

- 生成和管理账户地址
- 管理私钥或助记词
- 发起链上交易
- 连接 DApp
- 查看资产和交易记录

在传统互联网里，我们通常通过账号和密码登录平台；而在 Web3 中，用户更多是通过钱包证明“这个地址由我控制”。

### 2. 地址、私钥、助记词的关系

今天重点理解了几个概念：

- 地址：可以理解为 Web3 世界里的公开账号，可以分享给别人。
- 私钥：控制地址资产和权限的核心凭证，绝对不能泄露。
- 助记词：通常可以恢复整个钱包，拥有极高权限，也绝对不能泄露。
- 钱包：管理地址、私钥、助记词，并帮助用户和区块链交互的工具。

一个简单类比是：

- 地址像银行卡号，可以告诉别人。
- 私钥像银行卡密码加身份凭证，不能告诉别人。
- 助记词像恢复整个账户的最高权限凭证，更不能告诉别人。

### 3. Web3 安全意识

今天最重要的收获是：Web3 给了用户更高的自主权，但也要求用户自己承担更多安全责任。

需要特别注意：

- 不要把私钥、助记词、密码、API Key、Token 写进 GitHub。
- 不要随便在网页中输入助记词。
- 不要点击来历不明的钱包授权链接。
- 不要用真实资产钱包做学习实验。
- 初学阶段优先使用测试网进行练习。

这对我之后写学习笔记、提交代码和做项目都很重要。Web3 学习不只是理解概念，也要从第一天开始养成安全习惯。

### 4. 测试网和主网的区别

今天还初步了解到测试网和主网的区别：

- 主网是真实资产环境，任何操作都需要谨慎。
- 测试网适合学习和开发，测试币没有真实价值。
- 初学 Web3 时，应该优先在测试网中练习钱包、交易和合约交互。

## Key concepts

1. 钱包：管理私钥、地址和链上交互的工具，不是真正存放资产的容器。
2. 地址：公开账号，可以接收资产或作为身份标识。
3. 私钥：控制地址权限的核心凭证，泄露就可能失去资产控制权。
4. 助记词：恢复钱包的高权限凭证，必须离线、安全保存。
5. 测试网：适合学习和开发的链上环境，测试币没有真实价值。
6. 主网：真实资产环境，操作风险更高。
