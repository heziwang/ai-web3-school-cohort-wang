## 今日学习内容

### 1. AI 的进化

今天课程先回顾了 AI 能力的演进：

1. 早期 AI 更像“规则系统”，主要依赖人工设定规则。
2. 机器学习阶段开始从数据中学习模式。
3. 深度学习让模型能处理图像、语音、自然语言等复杂任务。
4. 大语言模型让 AI 具备更强的理解、生成、推理和对话能力。
5. Agent 阶段的重点不只是“回答问题”，而是能围绕目标调用工具、执行任务、保存上下文并持续协作。

我今天对 AI 的理解有一个变化：以前更容易把 AI 当成聊天工具，现在开始把它理解成可以连接工具、文件、GitHub、终端和工作流的“执行型助手”。

### 2. Hermes Agent 的核心能力

今天重点学习了 Hermes Agent 的能力和使用方式。我的理解是：Hermes Agent 不只是一个聊天机器人，而是一个可以在本地和多平台运行的 AI Agent 框架。

我记录到的核心能力包括：

- **工具调用**：可以读取文件、修改文件、运行命令、搜索资料、处理 Git 等。
- **跨平台使用**：可以通过终端、Telegram 等入口和 Agent 交互。
- **长期记忆**：可以记住稳定偏好和环境信息，减少反复说明。
- **Skills 技能系统**：把常用流程沉淀成可复用技能，让 Agent 下次处理类似任务时更稳定。
- **定时任务 / Cron**：可以让 Agent 在未来或周期性地执行任务。
- **多 Agent / 子任务委派**：复杂任务可以拆给子 Agent 并行处理。
- **GitHub / 文件工作流**：适合用来维护学习仓库、写笔记、提交和推送更新。

这部分让我意识到：AI × Web3 学习中，Agent 可以成为“学习助教 + 项目助手 + 复盘工具”，而不只是用来问概念。

### 4. Etherscan 学习记录

今天还学习了 Etherscan。我的核心理解是：

> Etherscan 是以太坊主网的区块浏览器和数据分析平台，可以用来查询链上交易、地址、区块、智能合约、Token、NFT、Gas 和网络状态等公开数据。

它不是交易所，也不是钱包，而是一个“链上数据查询入口”。

我今天记住的几个重点：

- **Transaction Hash / TX#**：一笔交易的唯一编号，可以用来查询交易详情。
- **Address**：链上地址，通常以 `0x` 开头，可以查看余额、交易记录、Token / NFT 持有情况等。
- **From / To**：交易的发起地址和接收地址。
- **Gas / Gwei**：交易需要支付的网络费用单位，`1 Gwei = 0.000000001 ETH`。
- **Block**：区块，链上交易会被打包进区块。
- **Smart Contract**：智能合约，链上自动执行规则的程序。
- **Verified Contract**：已验证源码的合约，更方便查看和理解合约逻辑。
- **Token / NFT**：Etherscan 可以查看 ERC-20、ERC-721、ERC-1155 等资产数据。
- **Pending Transactions**：还在等待确认的交易。
- **Internal Transactions**：合约内部调用产生的交易记录。
- **Token Approvals**：可以查看授权情况，和钱包安全相关。

今天我对 Web3 的一个重要理解是：链上行为是公开、可查询、可验证、可留痕的。Etherscan 就像进入以太坊公开账本的一扇门。

## 今日关键概念

1. **AI Agent**：不只是回答问题，而是能围绕目标调用工具、执行任务并持续处理上下文的 AI 系统。
2. **Hermes Agent**：一个可以运行在本地和多平台上的 Agent 框架，支持工具、记忆、技能、定时任务和多 Agent 协作。
3. **Etherscan**：以太坊区块浏览器，用来查看链上公开数据。
4. **Transaction Hash**：链上交易的唯一查询编号。
5. **Gas / Gwei**：以太坊交易费用相关概念。
6. **Smart Contract**：部署在链上的程序，用代码执行规则。
