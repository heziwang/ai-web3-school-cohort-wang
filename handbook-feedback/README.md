# Handbook Feedback 流程

学习中遇到以下内容时，记录到本目录：

- 概念不清楚或解释不够新手友好
- 页面结构建议
- 错别字、链接失效、资料过期
- 示例代码无法运行
- 希望新增的图示、FAQ、案例

## 命名规则

建议文件名：

```text
YYYY-MM-DD-short-topic.md
```

例如：

```text
2026-05-19-network-newbie-question.md
```

## 每条 feedback 建议包含

- Handbook 页面链接
- 问题类型：错别字 / 概念不清楚 / 链接失效 / 示例问题 / 结构建议 / 其他
- 问题描述
- 建议改法
- 来源：daily note、任务、实验或个人阅读
- 状态：draft / submitted / resolved

## 回流原则

- 保持可公开、可索引、可复盘。
- 不写隐私信息、secret、未公开会议内容。
- 优先给出具体链接和具体建议，而不是泛泛而谈。
